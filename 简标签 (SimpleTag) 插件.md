# 1 说明
这是 obsidian 的一个插件。
obsidian 的 tag 其实非常重量级，是针对整个文档的。而简标签则是针对一个句子的，侧重轻量级、随时捕捉灵感。
并没有上架 obsidian 社区，你可能需要 `brat` 等相关工具安装，我自己就是这样安装的。


# 2 特性
1. 语法简单：`%+标签+空格+内容+中文句号`
2. 支持筛选：类似 tasks、dataview，语法类似（高仿）
3. 支持高亮：类似 aDHL，设置面板可设置默认颜色、对不同标签进行设置

# 4 AI 写的插件说明

## 4.1 项目说明

**简标签**是一个 Obsidian 笔记插件，提供轻量级标签系统，用于在笔记中标记和查询关键内容。

**核心特性：**

- 使用 `%标签名 内容。` 格式标记关键句子
    
- 通过 `tag` 代码块查询和筛选
    
- 支持自定义标签样式（颜色）
    
- 实时同步，点击跳转
    

---

## 4.2 快速开始

### 4.2.1 标记标签

在笔记中使用 `%标签名 内容。` 格式：

```
%重要 学习数据结构和算法是编程的基础。
%待办 需要补充具体的代码示例。
%概念 面向对象编程的三大特性是封装、继承和多态。
```

**语法规则：**

- `%` 开头，后面紧跟标签名（不能有空格）
    
- 空格后写内容，以 `。` 结束
    
- 一行可以有多个标签
    

### 4.2.2 查询标签

使用 `tag` 代码块查询：

````
```tag
TABLE
FROM 重要
```
````

---

## 4.3 查询语法

```
[展示格式]
FROM [标签名...]
[WHERE 条件]
[SORT BY 字段 [ASC|DESC]]
[LIMIT 数量]
```

### 4.3.1 展示格式

- `TABLE` - 表格（默认），显示标签、内容、文件
    
- `LIST` - 有序列表，只显示内容
    
- `BULLET` - 无序列表，只显示内容
    

### 4.3.2 FROM 子句

支持一个或多个标签名（空格分隔，OR 逻辑）：

```
FROM 重要 紧急
```

### 4.3.3 WHERE 条件

- `content contains "文本"` - 内容包含
    
- `path contains "路径"` - 路径包含
    
- `tag = "标签名"` - 标签等于
    

### 4.3.4 SORT BY 排序

- `modified` - 修改时间
    
- `tag` - 标签名
    
- `path` - 文件路径
    

### 4.3.5 LIMIT 限制

```
FROM 重要
LIMIT 10
```

## 4.4 完整示例

### 4.4.1 基础查询

````
```tag
TABLE
FROM 重要
```
````

### 4.4.2 带过滤的查询

````
```tag

### 4.4.3 排序和限制

````

```
````

## 4.5 设置说明

### 4.5.1 默认样式

- **标签背景** / **标签文字** - 控制 `%标签名` 部分的颜色
    
- **内容背景** / **内容文字** - 控制 `内容。` 部分的颜色
    

### 4.5.2 自定义标签样式

1. 点击"+ 添加自定义样式"
    
2. 输入标签名（空格分隔可添加多个）
    
3. 点击 4 个颜色按钮设置颜色
    
4. 点击"添加"保存
    

### 4.5.3 预设颜色

- 点击任意颜色按钮打开预设面板
    
- 选择预设颜色或点击"+ 添加预设"创建新预设
    
- 右键自定义预设可删除

## 4.7 安装方法

1. 构建项目：`npm run build`
    
2. 复制以下文件到 Obsidian 插件目录：
    
    - `main.js`
        
    - `manifest.json`
        
    - `styles.css`
        
3. 在 Obsidian 设置中启用插件
